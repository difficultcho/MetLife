这是一个前后端分离的应用

1. 前端
  1.1 前端使用微信小程序开发
  1.2 小程序只有一个商品列表页面（双列商品列表），商品图片从后端接口获得
  1.3 用户点击某一款商品图片后触发：
    1.3.1 弹窗确认用户选择
    1.3.2 请求用户授权使用实名电话和微信昵称（getPhoneNumber、wx.getUserProfile）
    1.3.3 用户授权后，将用户身份、选择商品发送至后端保存，在页面展示适当提示感谢用户
    1.3.4 如果用户已经选择过商品（选择过商品的用户必然已经授权使用微信身份及电话），则提示用户已选择商品，询问是否要更换，确认后覆盖旧记录

2. 后端
  2.1 后端应用
    2.1.1 需要商品查询接口、用户信息提交接口
    2.1.2 需要配套的商品信息模型、用户信息模型
  2.2 后端数据库
    2.2.1 使用 mysql8.0 （服务器端已安装 mysql8.0）
  2.3 管理功能
    不需要开发管理功能，商品信息使用后端应用框架API上传，图片存储在静态目录；用户信息从微信小程序提交
  2.4 开发语言
    Python FastAPI

3. 部署
  3.1 部署在服务器 bce.kkmsee.com/metlife
  3.2 使用 github 的 ci/cd 进行部署
  3.3 使用 docker 容器进行部署（服务器端已安装 docker）

4. 其它
  4.1 前后端在同一个项目（repo）开发
